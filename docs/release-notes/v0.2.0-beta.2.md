# v0.2.0-beta.2

**Release date**: 2026-06-15  
**Release stage**: source/developer beta  
**Tag**: `v0.2.0-beta.2`  
**Commit**: `eb74d6c` (before version bump)

## Overview

Replyline v0.2.0-beta.2 is a source/developer beta release focused on release
engineering, security hardening, and documentation. No signed public Windows
installer is included — this release is intended for developers and testers who
build from source.

## What's new

### Release engineering

- **Release-freeze guard** protects critical paths (63 guardrail entries) from
  accidental drift. Advisory in normal CI, strict in `verify:full` and release
  workflows.
- **Canonical release guide** at `docs/engineering/release.md` — decision model,
  freeze semantics, packaging truth, and operator checklist.
- `pnpm beta:release-check` for pre-release readiness review.

### Security hardening

- Experimental bilingual feature fully gated behind `REPLYLINE_EXPERIMENTAL_BILINGUAL=1`
  env flag. Bilingual commands not callable when disabled.
- Release-on-tag workflow permissions reduced to job-level least privilege.
- CI workflows hardened: concurrency, timeouts, CI summary.
- SonarQubeCloud findings triaged (6 issues fixed).

### Code quality

- Frontend model split into 10 domain modules under `src/app/model/`.
- Tauri command registry centralized in `replyline_commands!` macro.
- Commands module converted to directory (`src-tauri/src/commands/`).
- Settings surface helpers extracted to `settings/settingsViewModel.ts`.

### Documentation

- **Provider setup guide** (`docs/product/provider-setup.md`) — OpenRouter,
  Groq, and custom OpenAI-compatible provider configuration with model tables
  and caveats.
- **Windows 11 source-beta smoke evidence** (`docs/beta-evidence/`) —
  sanitized partial smoke run documenting what passes and what remains
  pending verification.
- **Repository scorecard** (`docs/repo-scorecard.md`).

## Known limitations

- **Live provider evidence**: `pending verification` on the primary Windows 11
  test machine — Deepgram API key unavailable at time of release. Full
  capture→STT→LLM→card path not confirmed with live providers.
- **Cross-machine verification**: behaviour on Windows 10 and other Windows 11
  builds remains pending broader testing.
- **No signed installer**: unsigned artifacts remain internal only. Source
  build from `main` is the supported public onboarding path.
- **Experimental bilingual**: disabled by default; not part of the shipped
  product surface.

## Upgrade from v0.2.0-beta.1

```powershell
git checkout main
git pull origin main
pnpm install --frozen-lockfile
pnpm verify
```

Settings and credentials are preserved across source updates (stored in
`%APPDATA%\com.replyline.app\`).
