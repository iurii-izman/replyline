# Replyline v0.2.0-beta.3 — ContextPack Pivot

> **Date:** 2026-06-16
> **Type:** Source/developer beta (no signed public installer)
> **Decision:** Conditional Go
> **Defining change:** ContextPack ships as the new conversation context primitive

## What's new

### ContextPack — the new conversation context primitive

The biggest change in beta.3: **ContextPack** replaces the earlier Candidate Pack
concept with a single, universal conversation context primitive.

- **7 IPC commands**: list, save, delete, set active, clear active, get active, get status
- **UI panel**: Create, edit, delete, activate/deactivate packs directly from the app
- **Active chip indicator**: The idle screen shows which pack is active
- **Prompt injection**: Active ContextPack content is injected into LLM prompts with
  `Active conversation context:` heading and guardrails (do not treat as transcript,
  do not fabricate facts, prioritize transcript on conflict)

### Storage hardened

ContextPack storage (`context-packs.json`) is now resilient to file corruption:
- Corrupt JSON is automatically quarantined to `.corrupt.<timestamp>` backups
- The app recovers gracefully with an empty store instead of crashing
- Diagnostics track file existence, pack count, active pack presence, and corrupt backup count

### Answer profiles → universal answer styles

Internal answer profile IDs have been renamed from interview-specific to universal:
- `interview_default` → `work_default`
- `interview_concise` → `work_concise`
- `interview_star` → `work_star` (and so on)
- Old `interview_*` IDs still work transparently via alias mapping
- Prompt wording: «Answer style» / «Стиль ответа» replaces «Answer profile» / «Профиль ответа»

### Bilingual experimental track frozen

The bilingual interview mode is now fully gated behind a two-factor check:
- `REPLYLINE_EXPERIMENTAL_BILINGUAL=1` env flag (primary kill-switch)
- `bilingualInterviewEnabled: true` setting (user-facing opt-in)
- Both must pass. All 4 bilingual commands and the bootstrap enforce this gate.
- UI toggles and event listeners are hidden by default.

### Interview Mode repositioned

Interview Mode is now secondary — the idle state primary CTA is ContextPack
management. The interview mode remains accessible but no longer dominates the
home screen.

## Known limitations

- **Live provider evidence**: Automated probes measured. Manual ContextPack provider
  scenarios (ctx-live-01/02/03) pending — requires live desktop app run with synthetic
  audio and configured Deepgram API key.
- **Cross-machine smoke**: Windows 10 not tested.
- **No signed public installer** (S2): This is a source/developer beta. Build from
  source with `pnpm tauri build`. No code-signing certificate is available.
- **Provider keys required**: Users must configure their own Deepgram API key and
  OpenAI-compatible provider access.

## Breaking changes

- **Settings schema v10**: No breaking changes. All bilingual fields have safe defaults.
- **Answer profile IDs**: Old `interview_*` values in `settings.json` continue to work
  via transparent alias mapping. On next save, the canonical `work_*` ID will be written.

## Upgrade from beta.2

1. `git pull origin main`
2. `pnpm install --frozen-lockfile`
3. `pnpm tauri build` (or `pnpm tauri dev` for development)
4. Existing settings and data migrate automatically
