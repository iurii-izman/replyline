## Why beta.4

Beta.3 shipped ContextPack as the defining feature. Beta.4 shifts focus from
feature delivery to quality: UX polish, code maintainability, public docs, and
signing readiness. This is a stability release ‚Äî the product surface is stable,
the codebase is cleaner, and the public profile is stronger.

## What's new

### ContextPack daily-use UX

ContextPack panel improved for everyday use:
- **Empty state** now explains why ContextPack is useful and shows an example
- **Save notice** appears after creating or editing a pack
- **Active pack preview** shows compact content preview + character count
- **Duplicate** button for quick pack copying (no auto-activate)
- **Focus returns** to "New Context" button after save for keyboard flow
- **Editor placeholder** shows a synthetic context example

### Frontend maintainability ‚Äî locale split

`src/app/locale.ts` split from one ~880-line file into 7 domain modules:
- `locale/card.ts` ‚Äî card labels (gist, sayNow, nextMove, retry, etc.)
- `locale/interview.ts` ‚Äî interview card labels (73 keys)
- `locale/bilingualExperimental.ts` ‚Äî bilingual UI labels
- `locale/settings.ts` ‚Äî settings labels (137 keys)
- `locale/contextPack.ts` ‚Äî ContextPack labels
- `locale/common.ts` ‚Äî header, phase, setup, pipeline, errors, etc.
- `locale/index.ts` ‚Äî thin re-export hub (24 lines)

All public exports (`ui_ru`, `ui_en`, `UiStrings`, `getUi`) are preserved.
Imports from `"./locale"` continue to work without changes.

### Frontend maintainability ‚Äî surface splits

Two large component files split into focused sub-components:

**`MainSurface.tsx`** (1039 ‚Üí 290 lines): 8 components extracted to `src/app/main/`:
`SetupFocusState`, `IdleReadyState`, `ProcessingState`, `LiveAnswerCard`,
`InsightStrip`, `WorkspaceSidePanel`, `ActionDock`, `LiveAssistShell` + `helpers`.

**`SettingsSurface.tsx`** (994 ‚Üí 243 lines): 5 sections extracted to `src/app/settings/`:
`OverviewSection`, `SpeechSection`, `LlmSection`, `HotkeySection`, `ReportsSection`.

No behavior changes. All `data-testid`, classes, aria attributes, and handlers preserved.

### Rust maintainability ‚Äî command domain split complete

`src-tauri/src/commands/mod.rs` reduced from 457 to 28 lines. All 12 command domains
now in separate modules:
`bootstrap`, `capture`, `context`, `context_pack`, `diagnostics`, `interview`,
`bilingual_experimental`, `runtime_checks`, `secrets`, `settings`, `shared`, `tray_window`.

Central registry (`registry.rs`) updated with direct module paths. IPC contract
checker extended to validate all 12 source files.

## Known limitations

- **Live provider evidence**: All automated gates green. Manual ContextPack provider
  scenarios (ctx-live-01/02/03) remain pending ‚Äî requires live desktop app run with
  synthetic audio and configured Deepgram API key.
- **Cross-machine smoke**: Windows 10 not tested.
- **No signed public installer** (S2): This is a source/developer beta. Build from
  source with `pnpm tauri build`. No code-signing certificate is available.
  Signing readiness plan is documented in `docs/engineering/windows-signing-readiness.md`.
- **Provider keys required**: Users must configure their own Deepgram API key and
  OpenAI-compatible provider access.
- **Live answer quality evaluation**: Design complete, implementation blocked by
  provider key availability.
