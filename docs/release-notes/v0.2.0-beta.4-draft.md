# Replyline v0.2.0-beta.4 — Quality, Stability & Public Trust

> **Date:** 2026-06-18 (updated)
> **Type:** Source/developer beta (no signed public installer)
> **Decision:** Conditional Go — see checklist below
> **Theme:** Quality hardening, maintainability, public trust. No new features.

## Why beta.4

Beta.3 shipped ContextPack as the defining feature + Product Experience Hardening
(UX score 85→88). Beta.4 shifts focus from feature delivery to public trust:
documentation, release packaging discipline, signing readiness, and honest public
posture. The product surface is stable. The codebase is cleaner. The public profile
is stronger.

## What's new

### Product Experience Hardening (UX 85→88)

ContextPack daily-use UX polished for real-world flow:

- **Idle screen** now explains ContextPack value proposition (`idleValueHint`)
- **Active context chip** uses ghost styling (not danger) for disable — less noisy
- **Answer "Скажи сейчас" area** has enhanced visual treatment (gradient, padding, larger text)
- **Error recovery** shows actionable 3-step guidance instead of generic message
- **ContextPack editor** has explicit label associations (`for`/`id`) for screen readers
- **Settings nav** visually separates essential sections from advanced
- **Answer card** has `role="region"` + `aria-label` for screen reader grouping
- 189 UI tests cover the hardened paths

### Frontend maintainability

- `locale.ts` split from one ~880-line file into 7 domain modules
- `MainSurface.tsx` (1039 → 290 lines): 8 components extracted to `src/app/main/`
- `SettingsSurface.tsx` (994 → 243 lines): 5 sections extracted to `src/app/settings/`

### Rust maintainability

- `commands/mod.rs` reduced from 457 to 28 lines
- All 12 command domains in separate modules

### Public trust package (new)

- **Release artifact manifest** (`docs/engineering/release-artifact-manifest.md`) —

  SmartScreen expectation note, rollback plan, public wording rules
- **Windows signing readiness** (`docs/engineering/windows-signing-readiness.md`) —

- **Provider runtime evidence matrix** — updated for 2026-06-18: 189 TS tests, 265 Rust tests

### Documentation & onboarding

- Desktop E2E smoke scaffold (5 checks)

## Known limitations

- **Live provider evidence**: All automated gates green. All manual provider scenarios
  (ctx-live-01 through ctx-live-06) blocked — `DEEPGRAM_API_KEY is missing`.
  Exact unblock checklist in `docs/beta-evidence/contextpack-live-runtime-2026-06-18.md`.
- **Cross-machine smoke**: Windows 10 not tested.
- **No signed public installer** (S2): This is a source/developer beta. Build from
  source with `pnpm tauri build`. Code-signing certificate not acquired.
  Signing readiness plan: `docs/engineering/windows-signing-readiness.md` (7/12 gates green).
- **Provider keys required**: Users must configure their own Deepgram API key and
  OpenAI-compatible provider access.
- **SmartScreen reputation**: Not applicable until first signed build exists.
