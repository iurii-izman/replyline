# Replyline v0.2.0-beta.2 Draft

This draft prepares the release readiness gate for the next beta publish step.
It is not an actual release event and does not authorize binaries, GitHub Release
creation, or installer publication.

## Highlights

- Release readiness gate added as `pnpm beta:release-check`.
- Release checklist documents the split between automated gates and manual review.
- Sanitized smoke report remains the supported issue attachment path.
- Release notes now separate setup guidance, privacy notes, and troubleshooting.

## Setup Instructions

1. Read [Engineering release guide](../engineering/release.md).
2. Run `pnpm beta:doctor` and fix any WARN/FAIL rows first.
3. Run `pnpm beta:smoke-report` and keep the generated sanitized report.
4. Run `pnpm beta:release-check` for the full readiness summary.
5. Review [Release checklist](../release-checklist.md) before claiming beta readiness.

## Manual Review Checklist

- first-run setup UX audit
- keyboard-only accessibility
- clean install on a separate Windows profile
- Deepgram setup path
- OpenRouter setup path
- custom OpenAI-compatible path
- Candidate Pack preparation
- Interview report export
- uninstall/clear local data note

## Known Limitations

- This draft does not create a public release or publish binaries.
- Clean install evidence must be collected manually on Windows.
- Keyboard-only accessibility and first-run UX are manual review items.
- Cross-machine and cross-provider behavior remains `pending verification` until tested.
- Provider latency and availability still vary by environment and account setup.

## Privacy Notes

- Keep API keys, bearer tokens, raw transcripts, raw prompts, and raw Candidate Pack values out of public issues.
- Use the redacted export path when sharing interview report content outside the repo.
- Treat `artifacts/beta-smoke-report/smoke-report.md` as sanitized but still review it before posting.

## Troubleshooting

- If `beta:doctor` warns about toolchains or runtime components, resolve those first.
- If `beta:smoke-report` fails, attach the sanitized report and the specific failing command output.
- If release-freeze strict mode fails, keep the changed files inside guardrails before release handoff.

## How to Report an Issue

1. Run `pnpm beta:smoke-report`.
2. Attach `artifacts/beta-smoke-report/smoke-report.md` and `artifacts/beta-smoke-report/smoke-report.json`.
3. Summarize the problem briefly in the issue body.
4. Do not paste raw logs, raw provider responses, or raw local paths.
5. Link this draft plus [Engineering release guide](../engineering/release.md) when filing release readiness follow-up.
