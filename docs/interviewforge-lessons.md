# InterviewForge Lessons for Replyline

These are the strongest ideas worth borrowing from InterviewForge without dragging Replyline into interview-specific product scope.

| Rank | Practice | Score / 100 | Recommendation | Why |
|---|---|---:|---|---|
| 1 | Separate verification lanes | 98 | Adopt now | Prevents fake confidence from one green command. |
| 2 | Settings separate from secrets | 97 | Adopt now | Correct trust baseline for a desktop AI tool. |
| 3 | Runtime evidence bundles | 95 | Adopt now | Creates machine-local proof instead of hand-wavy “works on my machine”. |
| 4 | Readiness before feature claims | 93 | Adopt now | Forces provider path proof before product promises. |
| 5 | Copy and trust discipline | 91 | Adopt now | Keeps the product away from stealth / therapy / fake-magic framing. |
| 6 | Benchmark label honesty (`target / measured / pending`) | 89 | Adopt now | Stops invented latency claims. |
| 7 | Explicit runtime bring-up runbook | 86 | Adopt now | Makes native Windows/runtime proof repeatable. |
| 8 | Prompt-contract corpus | 84 | Adopt now | Keeps output shape stable while prompts evolve. |
| 9 | Lightweight release readiness checklist | 78 | Adopt soon | Useful before alpha distribution, but should stay lean. |
| 10 | In-app inspect / exported diagnostics | 72 | Later | Very useful with external testers, but not core for the first tight MVP loop. |

## Adopt now

- verification lanes
- settings vs secrets split
- runtime evidence bundles
- readiness before claims
- copy guard
- benchmark honesty
- runtime bring-up doc
- prompt-contract corpus

## Adopt soon

- release readiness checklist

Keep this small. Replyline does not need the full InterviewForge release bureaucracy.

## Later, only if testers justify it

- in-app inspect panel
- weekly metrics ritual
- optional observability dashboards

Those are useful once the product has active external testers or repeated runtime debugging needs.

## Explicitly do not borrow

- multi-shell product architecture
- interview domain models
- full review / export flows
- transcript-heavy UI
- saved session sprawl as a default
