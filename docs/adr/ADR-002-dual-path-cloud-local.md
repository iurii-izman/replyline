# ADR-002: Dual-path cloud/local

## Статус

Accepted

## Дата

2026-04-10

## Контекст

Зависимость только от облачных LLM создаёт single point of failure и повышает риски простоя при сетевых или провайдерских сбоях.

## Решение

Использовать dual-path через LiteLLM: cloud-путь (`review-default`) как основной и fallback на local-путь (`Ollama/llama3.2`).

## Последствия

- Работа стека в офлайн/ограниченных сетевых сценариях.
- Потенциальное снижение затрат при переключении части нагрузки на локальные модели.
- Разница в качестве и консистентности результатов между cloud и local моделями.

## Альтернативы

- Только облако (отвергнуто): высокий риск отказа при недоступности внешнего провайдера.
- Только локально (отвергнуто): ограничения по качеству и функциональности моделей.
